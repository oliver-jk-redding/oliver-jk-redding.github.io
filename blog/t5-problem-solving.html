Problem Solving

14 February 2016

Overcoming Problems in Code

During the coding of my racing game I encountered a number of problems with my code. I find problems can usually fall into two main categories:

  • Syntax errors: These are problems in the actual grammar of your code e.g. putting a semi-colon in the wrong place or forgetting to use a capital letter. These errors can actually make the code unreadable for your computer, stopping the whole website from working, but are usually easy to solve.
  • Logic errors: These are problems to do with the output of your code. The code might be grammatically correct and make sense to the computer, but the code produces an outcome different to the one you were expecting. This is usually because you wrote the code to achieve a particular outcome, but didn't follow through the logic properly and so the outcome is actually different to what you prepared for. These can be harder to detect and fix.

If I encounter an error in my code, the first thing I do is look for error messages in the console. This can resolve the issue quickly as the console will usually show the line of code it is getting stuck on and tell you which line in the document it is. This can be particularly effective for solving syntax errors, for example finding that extra closing bracket you forgot to delete. It can also solve some logic errors, for example it might be that a particular variable has been undefined and the console can quickly identify which variable that is.

If that doesn't solve it, things can take a bit longer. It's time to isolate the problem. I add console.log() statements at various key points throughout the code to check that the program is reaching a desired section of code and that the values are as expected. A console.log() statement is a line of code that tells the program to print a specified output. This is especially useful with condition statements. Sometimes if a condition looks like it is being met, but the action in that condition is not happening, it can be useful to stick in a console.log() statement to see if the problem is with the action code or the condition code. If the console.log() statement fails to fire, then I know the problem is with the condition, etc.

Console.log statements can also be useful to check variables have the expected values. I might think a particular variable has the value '6' but in fact it has a value of '0'. Console.log() statements can quickly identify such problems.

Another trick is to 'comment' sections of more complex code and try running the program with more simplified code. E.g. if I can't get a function like checkKeyPressed() to fire off another complex function, updatePlayerPosition(), I need to know if the problem is with the checkKeyPressed() function or the updatePlayerPosition() function. So I will 'comment' or deactivate the checkKeyPressed() function and write a simpler version that instead of firing another function, does nothing more complicated than logging the key pressed to the console. Depending on the result I will know in which function the problem lies.

Sometimes the problem lies just with me not knowing the right code in which case I will comb google, w3schools, stackoverflow, and other useful sites to look for the right code or the correct way to implement that code. I'll often steal lines of code from these sites and paste it into my own code then modify it to suit my needs.

In a nutshell that's what I do. I have learned that working on a tired brain doesn't help either. The other night I was up until 1am trying to solve a problem which was as simple as writing 'keyCode' with a capital 'C' instead of a little 'c'.

It is worth having these techniques. There are few things more satisfying than finally solving a complex and frustrating problem and seeing your project advance to the next step.

My experience of journaling

I used a couple of the prompts provided. I initially struggled to think what to write but I did as suggested and continued to write 'I don't know' until some thoughts came to mind. I started writing about an idea I had had lying in bed that morning and my fears and hopes around that idea. Doing that, I was able to better process my thoughts and it prompted me afterwards to delve into that idea further. I wouldn't say the benefits were major or life-changing but it was an interesting exercise.

What did I take away from the "My Emotions are Not me" section?

This resonated with me quite strongly as I had only an hour before had an argument with my wife. During a pause in the argument, I sat thinking of the next angry response I could give, but then suddenly became aware I was angry. I realised in that moment that anger was just an emotion and could be controlled. I thought, I don't like being angry with my wife and it would be nice to return to a state of not being angry with her. So I resolved to dismiss my anger. But it was not so easily done. I still felt mad about what she had done. Further to that, I felt annoyed about me having to be the one to dismiss my anger and not her. I knew she would still be mad at me. I thought "why do I have to control my emotions when she doesn't?" I then thought "if I dismiss my anger now, should I always dismiss it? Should I never allow myself to become or stay angry?" I didn't like that thought very much as there are times when I see injustice being done and I don't respond because I can be quite calm and slow to anger and then I walk away thinking "I wish I had gotten angry and responded to that." I came then to another insight. Anger is not in itself a bad emotion. It has a place and a purpose. Biologically, that purpose was to help us survive, to help us fight competitors. In today's world it can help us to set wrongs aright and fight injustice. Not that my wife was being unjust in that sense. But I resolved that my anger had a place and a right to be felt. What mattered was not dismissing it, but being mindful of it and in control of it. I didn't make an angry response to further inflame the situation. I was still mad, but I resolved to control that anger, feel it, be mindful of it, and let it pass in its own time. In the next half hour, my anger had passed, so had my wife's, and we both moved on with our lives.
