Javascript Basics

16 January 2016

How does Javascript compare to HTML and CSS?

Javascript is different from HTML and CSS because it has a different purpose and consequently a different architecture. Whereas HTML and CSS are to do with structuring content on a webpage by organising that content into tags which determine the front-end appearance of a website, Javascript operates one layer back, by determining how objects on a website interact with each other and with the user.

Javascript is an 'object-oriented' programming language so it is based on the idea of creating objects which have properties and which can be manipulated and interact with other objects. It is similar to HTML and CSS in the sense that is utilises prototype objects which are similar to classes to organise content. In HTML and CSS, classes are used to define a group of tags which we want given the same style. In Javascript prototype objects are used to define a group of objects which we want to have the same properties. But the similarity ends there. Objects behave quite differently from tags in that the properties do not have a visible effect on the object's apperance like styles do; they are purely abstract concepts. For example, a penguin object can have properties describing its wing-span, colour, and species, but these are purely abstract descriptions that only have meaning insofar as they relate to other abstract objects.

An Explanation of Control Flow and Loops Using an Example Process from Everyday Life

Control flow statements look at a set of conditions and make a decision on what action to take based on the condition met. It is a good formula to use if a choice needs to be made. For example, if I am trying to decide what to have for breakfast, I might go through a process like this:

'If I am hungry AND feeling thin, I will have cocopops for breakfast.' 'Or if I am hungry AND feeling thin AND feeling hungover, I will have bacon and eggs for breakfast.' 'Or if I am not feeling hungry OR if I am feeling fat, I will have a hard-boiled egg for breakfast.'

A loop statement is used to repeat an action until a particular condition has been met. For example, if I am eating a packet of potato chips, I might go through a process like this.

'While I am hungry, eat another chip. When I am no longer, hungry, stop.'

The difference between accessing data from arrays and object literals

Objects contain descriptions called properties. Properties are a series of variables which can contain data in the form of values. These values can be accessed directly by referencing the object name, placing a '.' after it, and writing the name of the variable, but this only works if the variable is public. If it is private, a public method will have to be defined for the object in order for the value to be accessed. Similarly private values can only be changed if a public method provides for it. Another limitation is that you need to know the name of the variable in order for the value to be accessed.

An array stores values in a series of indices. These can be accessed by providing the name of the array and then putting the index of the variable in square brackets. This is very easy making array data more easily accessible than object literals, but the downside is that you need to know the index where the value is located. You can easily search through the indices until you find the value you are looking for but with especially large arrays, this can take time.

What are functions and why are they useful?

Functions are lines of code assigned to a name. When one wants those lines of code to be implemented one only has to call the function by invoking the name. This is very useful because it avoids the need to repeat lines of code whenever the same action needs to be repeated. This saves time in the writing of the code and makes the code less cluttered. It is also in keeping with the programming principle of D.R.Y. (Don't Repeat Yourself).
